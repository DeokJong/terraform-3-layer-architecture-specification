# Legacy Convention

- Do not rename existing assets immediately without clear operational value.
- Reinterpret existing assets using the new ownership model first.
- Apply the Convention to all newly introduced resources and Contracts.
- Prefer parallel publication and gradual migration over forced rename or cutover.

## Legacy handling rule

Legacy is allowed as a starting point, but future design and migration should move toward clearer ownership, Contract publication, and safer Workspace boundaries.
