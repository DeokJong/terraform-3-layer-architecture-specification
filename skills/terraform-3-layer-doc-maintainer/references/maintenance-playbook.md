# Maintenance Playbook

## High-value improvements

Prefer these improvements when strengthening the documentation:

- clarify architecture views
- add decision matrices
- add reusable templates
- add failure and migration scenarios
- improve navigation and reading order
- reduce duplicated rules across files
- make source-of-truth ownership explicit

## Source-of-truth discipline

- root specification defines the canonical architecture meaning
- root conventions defines implementation rules
- `docs/` republishes and reorganizes for reading
- registry tracks status, scope, and next actions across all document families

## Progress tracking model

Use these status values in the registry:

- `stable`
- `needs-strengthening`
- `in-progress`
- `planned`

Use short next actions focused on concrete upgrades, not vague goals.
